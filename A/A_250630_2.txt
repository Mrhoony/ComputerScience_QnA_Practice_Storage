1. OSI 7계층이란?
->
개방형 시스템 상호 연결 모델 (Open Systems Interconnection Reference Model)
OSI 7계층 관점에서 네트워킹이란 데이터가 계층마다 조립/분해되어 서로 주고받는 것이다.

[ L1, 물리 계층 (Physical Layer) ]
기능: 비트 단위의 데이터를 전기적, 광학적 신호로 변환하여 물리적인 전송 매체(케이블, 광섬유, 무선 등)를 통해 전송합니다. 단순히 데이터를 전달하는 역할만 하며 오류 제어나 흐름 제어 기능은 없습니다.
전송 단위: 비트 (Bit)
관련 장비: 허브, 리피터, 케이블, 모뎀

중요성: 클라이언트 프로그래머가 이 계층을 직접적으로 다루는 경우는 거의 없습니다. 하지만 네트워크 연결이 원활하지 않을 때 가장 기본적인 문제를 진단하는데 필요한 배경 지식을 제공합니다. 게임 실행 중 네트워크 연결이 끊어지거나 불안정할 때 이 계층의 문제를 간접적으로 파악하고 사용자에게 적절한 안내를 제공하는데 활용될 수 있습니다.


[ L2. 데이터 링크 계층 (Data Link Layer) ]
기능: 물리 계층을 통해 전송되는 비트들을 프레임이라는 논리적인 단위로 묶어 송수신합니다. 인접한 두 장치 간의 신뢰성 있는 데이터 전송을 담당하며 MAC 주소를 사용하여 통신하고 오류 감지 및 수정, 흐름 제어를 수행합니다.
전송 단위: 프레임 (Frame)
관련 장비: 브릿지, 스위치


[ L3. 네트워크 계층 (Network Layer) ]
기능: 데이터를 패킷(Packet) 단위로 묶어 송신측에서 수신측까지 가장 효율적인 경로를 찾아 전송합니다. IP 주소를 사용하여 다른 네트워크 간의 통신을 관리하며 라우팅(경로 설정) 기능을 수행합니다.
전송 단위: 패킷 (Packet)
관련 장비: 라우터

중요성: 클라이언트 프로그래머가 직접 IP 주소를 다루는 경우는 많지 않지만 게임 서버의 IP 주소로 통신을 시도하고 라우터를 거쳐 서버까지 도달한다는 개념은 필요합니다. 특히 공용 IP/사설 IP, NAT(Network Address Translation) 같은 개념은 멀티플레이어 게임에서 흔히 발생하는 연결 문제를 이해하는데 도움이 됩니다.
UE: UE 클라이언트는 서버 IP 주소를 통해 연결을 시도합니다. 연결이 되지 않을 때 단순히 네트워크 문제라고 생각하기보다 IP 주소 문제나 방화벽 문제일 수 있음을 추록할 수 있는 기초 지식이 됩니다.


[ L4. 전송 계층 (Transport Layer) ]
기능: 송신자와 수신자 간의 신뢰성 있는 데이터 전송을 보장합니다. 데이터를 세그먼트(Segment) 단위로 쪼개고, 포트 번호를 사용하여 각 프로세스에 데이터를 정확히 전달합니다. 오류 검출 및 복구, 흐름 제어, 다중화/역다중화 기능을 수행합니다.
주요 프로토콜: TCP (Transmission Control Protocol), UDP (User Datagram Protocol)
전송단위: 세그먼트 (Segment, TCP), 데이터그램 (Datagram, UDP)

중요성: 게임 네트워크 통신의 핵심입니다. 어떤 프로토콜을 사용할지에 따라 게임의 성능과 안정성이 크게 달라집니다.
TCP: MMORPG의 캐릭터 위치, 인벤토리 정보, 스킬 사용 결과 등 정확성이 중요한 데이터 전송에 사용됩니다. 채팅, 로그인, 상점 이용 등 데이터 손실이 치명적인 경우에 적합합니다.
UDP: FPS, MOBA와 같은 실시간 액션 게임에서 데이터의 신뢰성보다 데이터의 실시간성이 중요시되는 경우에 사용됩니다. 캐릭터 위치, 이동 정보, 애니메이션 상태 등의 데이터가 필요에 따라 UDP로 송수신될 수 있습니다. 약간의 데이터 손실은 게임 플레이에 큰 지장을 주지 않지만 지연이 치명적인 경우에 적합합니다.
UE: UE는 기본적으로 자체 네트워크 코드를 가지고 있으며 UDP 기반으로 최적화된 통신을 제공합니다. (Reliable -> RUDP)
핵심: 게임의 종류와 특성에 따라 TCP와 UDP 중 어떤 프로토콜이 적합한지 판단하고 각 프로토콜의 장단점을 이해하여 네트워크 코드를 최적화하는 능력이 요구됩니다. 패킷 손실, 지연 시간(Latency), 대역폭(Bandwidth) 등 네트워크 환경에 대한 이해가 필수적입니다.

TCP의 세그먼트는 신뢰성을 보장하기 위한 다양한 기능(연결 설정/해제, 순서 번호, ACk 응답, 재전송 등) 때문에 데이터 전송에 더 많은 오버헤드가 발생하고 UDP의 데이터그램은 이러한 기능이 없어 TCP에 비해 헤더가 매우 간단하고 오버헤드가 적습니다.


[ L5. 세션 계층 (Session Layer) ]
기능: 통신하는 응용 프로그램 간의 세션을 설정, 유지, 종료하는 역할을 합니다. 연결 관리, 동기화, 대화 제어 등을 수행하여 데이터 전송의 효율성과 안정성을 높입니다.


[ L6. 표현 계층 (Presentation Layer) ]
기능: 데이터를 응용 프로그램이 이해할 수 있는 형식으로 변환합니다. 데이터의 암호화 및 복호화, 압축 및 압축 해제, 문자 인코딩(ASCII, EBCDIC 등) 변환 등을 담당합니다.


[ L7. 응용 꼐층 (Application Layer) ]
기능: 사용자와 직접 상호작용하는 최상위 계층입니다. 이메일, 웹 브라우징, 파일 전송 등과 같은 응용 서비스가 이 계층에 해당하며 사용자가 네트워크에 접근할 수 있도록 서비스를 제공합니다.
주요 프로토콜: HTTP, FTP, SMTP, DNS 등

중요성: 게임 클라이언트 프로그래머의 주요 작업 계층으로 게임의 핵심 로직 (캐릭터 이동, 스킬 사용, 아이템 획득, 채팅 등)은 모두 이 계층에서 정의된 통신 프로토콜을 통해 서버와 주고받습니다.
UE: UE의 RPC 시스템이나 Replication 시스템이 이 계층과 밀접하게 관련되어 있습니다. 클라이언트에서 서버로 특정 액션을 요청하거나 서버에서 클라이언트로 게임 상태를 동기화하는 과정이 응용 계층 수준에서 이루어집니다.
핵심: 게임 플레이에 필요한 모든 데이터와 그 데이터가 어떤 의미를 가지는지 어떻게 주고받아야 하는지를 설계하고 구현하는 것이 이 계층의 역할이며 클라이언트 프로그래머의 핵심 임무입니다.



2. TCP/IP 4계층이란?
->
OSI 7계층 모델이 이론적인 네트워킹의 계층 구조라면 TCP/IP 4계층 모델은 실질적으로 현재 인터넷에서 가장 널리 사용되는 네트워크 통신 모델입니다. OSI 7계층을 계층의 주체, 책임의 범위에 따라 간소화한 형태로 실제 프로토콜 스택과 밀접하게 연관되어 있습니다.

[ L1. 네트워크 액세스 계층 (Network Access Layer) ]
OSI: OSI 모델의 L1 + L2, 물리 계층과 데이터 링크 계층의 기능을 모두 포함합니다.

주요 역할.
물리적 전송: 데이터를 전기적, 광학적 또는 무선 신호로 변환하여 물리적인 매체를 통해 전송하고 수신하는 역할을 담당합니다.
MAC 주소 사용: 같은 네트워크(LAN) 내에서 장치들을 식별하기 위해 MAC(Media Access Control) 주소를 사용합니다.
프레임 구성: IP 패킷을 물리적으로 전송 가능한 프레임 단위로 캡슐화합니다.
오류 감지: 물리적인 전송 과정에서 발생할 수 있는 오류를 감지하는 기능을 제공합니다.

주요 프로토콜 및 기술: 이더넷(Ethernet), Wi-Fi(IEEE 802.11), PPP(Point-to-Point Protocol) 등
관련 장비: 허브, 리피터, 스위치, 네트워크 인터페이스 카드(NIC)


[ L2. 인터넷 계층 (Internet Layer) ]
OSI: OSI 모델의 L3, 네트워크 계층에 해당합니다.

주요 역할.
논리적 주소 지정 (IP 주소): 데이터를 보내려는 장치를 IP 주소를 사용하여 식별합니다. 전 세계의 모든 장치는 고유한 IP 주소를 가집니다.
패킷 라우팅: 서로 다른 네트워크에 있는 장치들 간에 데이터(패킷)를 전송하기 위한 최적의 경로를 결정하고 해당 경로를 따라 패킷을 전달합니다. (라우팅)
패킷 구성: 전송 계층으로부터 받은 데이터를 패킷(Packet) 단위로 캡슐화합니다.

주요 프로토콜.
IP (Internet Protocol): TCP/IP 모델의 핵심으로 데이터 패킷 주소 지정 및 라우팅을 담당합니다.
ARP (Address Resolution Protocol): IP 주소를 MAC 주소로 변환하는데 사용됩니다.
ICMP (Internet Control Message Protocol): 네트워크 오류 보고 및 진단(ping 명령어 등)에 사용됩니다.

관련 장비: 라우터, L3 스위치


[ L3. 전송 계층 (Transport Layer) ]
OSI: OSI 모델의 L4, 전송 계층에 해당합니다.

주요 역할.
종단 간(End-to-End) 통신: 송신자와 수신자 간의 데이터 전송을 담당합니다. 특정 응용 프로그램(프로세스)에 데이터를 정확히 전달하기 위해 포트 번호를 사용합니다.
신뢰성/비신뢰성 보장: 데이터의 신뢰성을 보장할지(TCP) 또는 빠르게 전달할지(UDP)를 결정합니다.
오류 제어 및 흐름 제어: TCP의 경우 데이터 손실이나 중복 없이 순서대로 전달되도록 보장하고 송수신 장치의 처리 속도 차이로 인한 문제(버퍼 오버플로우)를 방지합니다.
세그먼트/데이터그램 구성: 응용 계층의 데이터를 TCP의 세그먼트 또는 UDP의 데이터그램 단위로 분할하고 캡슐화합니다.

주요 프로토콜.
TCP: 연결 지향적이며 신뢰성 있는 데이터 전송을 보장합니다.
UDP: 비연결 지향적이며 빠르고 효율적인 데이터 전송에 초점을 맞춥니다.


[ L4. 응용 계층 (Application Layer) ]
OSI: OSI 모델의 L5 + L6 + L7, 세션 계층과 표현 계층과 응용 계층의 기능을 모두 포함합니다.

주요 역할.
사용자와의 상호작용: 사용자에게 직접 네트워크 서비스를 제공하며 응용 프로그램이 네트워크에 접근할 수 있도록 합니다.
데이터 형식 변환 및 암호화: 데이터를 응용 프로그램이 이해할 수 있는 형식으로 변환하고 필요에 따라 암호화/복호화를 수행합니다.
세션 관리: 응용 프로그램 간의 통신 세션을 설정하고 유지하며 종료하는 역할을 합니다.

주요 프로토콜.
HTTP (Hyper Text Transfer Protocol): 웹 브라우징
FTP: 파일 전송단위
SMTP (Simple Mail Transfer Protocol): 이메일 송신
POP3/IMAP (POST Office Protocol 3/Internet Message Access Protocol): 이메일 수신
DNS (Domain Name System): 도메인 이름을 IP 주소로 변환
SSH (Secure Shell): 원격 접속  
Telnet: 원격 터미널 접속
